# Jawafdehi Corruption Case Script Generator

## Overview

This skill generates two distinct types of documentary-style scripts in **Nepali language only** (no Hindi words) from Jawafdehi corruption case summaries:

1. **PRESENTATION Scripts** — 15-20 minute interactive format for Zoom/in-person presentation with multiple presenters, clearly marked segment transitions, presenter cues, and Q&A integration points
2. **REELS/VOICEOVER Scripts** — Social media format (<230 seconds) with catchy phrasing, visual cues for video team ([B-ROLL], [GRAPHICS]), engagement hooks, and strategic pauses

Both formats maintain journalistic integrity, presume innocence throughout, and include recommended 2-3 second pauses at natural break points.

### ⚠️ IMPORTANT: Always Ask Format & Length First
The skill **always starts by asking what the script is for (format) and how long it should be (length)**. Only after those are confirmed does it gather case details. This ensures the script structure is appropriate from the start.

## Initial Elicitation: Ask Format & Length First

When Claude receives a request to generate a Jawafdehi script, **the first step is ALWAYS to ask what the script is for and its desired length**. This determines which use case and structure to follow.

### PRIMARY QUESTIONS (Always Ask These First)

```

२. स्क्रिप्टको लम्बाइ कस्तो हुनु चाहिन्छ?
   (What length should the script be?)
   
   IF PRESENTATION SELECTED:
   - १५ मिनेट
   - २० मिनेट
   - (Other: specify)
   
   IF REELS/VOICEOVER SELECTED:
   - १ मिनेट (60 सेकेन्ड)
   - १.५ मिनेट (90 सेकेन्ड)
   - २ मिनेट (120 सेकेन्ड)
   - २.५ मिनेट (150 सेकेन्ड) [Default: under 230 seconds max]
```

### SECONDARY QUESTIONS (After Format & Length Are Confirmed)

Once you know the format and length, ask:

```
३. तपाईंको अनुसन्धान/मामिला कस्तो छ?
   (What case are we covering?)
   
   प्रदान गर्नुहोस् / Provide:
```

### TERTIARY QUESTIONS (Format-Specific)

**IF REELS/VOICEOVER (<230 seconds):**

## Use Case 1: PRESENTATION SCRIPT (15-20 minutes)

### Purpose
Multi-presenter format for interactive Zoom/in-person presentation with consistent structure regardless of who presents, designed to accommodate Q&A sessions.

### Target Length
- **Total: 15-20 minutes** of spoken content
- **Structure varies by case complexity** (see below)
- **Includes explicit Q&A markers** where audience can interrupt
- **Presenters can switch at segment boundaries**

### Core Structure (Flexible Based on Complexity)

#### Simple Cases (1-2 accused, 2-3 allegations)
```
1. Title/Opening (2-3 min)
2. Case Summary & Context (3-4 min)
3. Key Allegations (4-5 min)
4. [Q&A PAUSE #1 - 2 min]
5. Timeline & Investigation (3-4 min)
6. Verdict/Status (2-3 min)
7. [Q&A PAUSE #2 - 2 min]
8. Implications & Call to Action (1-2 min)
```

#### Complex Cases (3+ accused, 4+ allegations)
```
1. Title/Opening (2-3 min)
2. Context & Overview (3-4 min)
3. Accused Profiles Section (3-4 min)
   └─ [Segment transition: "अब हामी मुख्य आरोपितहरू को बारे मा कुरा गरौँ"]
4. Specific Allegations by Accused (5-6 min)
   └─ [Clear markers between each accused: "पहिलो आरोपित..."]
5. [Q&A PAUSE #1 - 2-3 min]
6. Timeline & Investigation Process (3-4 min)
7. Verdict/Legal Status (2-3 min)
8. [Q&A PAUSE #2 - 2-3 min]
9. Impact & Public Accountability (2 min)
10. Call to Action (1 min)
```

```

कारण [IMPACT: पारदर्शिता, सार्वजनिक जागरूकता, न्याय आदि]।

[पज (2-3 सेकेन्ड)]
```

## Critical Constraints (Both Formats)

### Language
- **Nepali language ONLY** — No Hindi words (e.g., "अपराध" not "अपराधी", use "गलत काम" for misconduct)
- **No heavy jargon** — Avoid legal terms; use plain Nepali
  - ❌ अपचार → ✓ गलत काम
  - ❌ कर्तव्य उल्लंघन → ✓ कर्तव्य पूरा गरेनन्
  - ❌ मिसएप्लीकेशन → ✓ पैसा गलत जागोमा गयो
- **Audience:** 12-80 years old; assume general education level

### Presumption of Innocence (NON-NEGOTIABLE)
- **ALWAYS use:** 'आरोप' (allegation), not assumptions of guilt
- **NEVER use:** 'उनले गरे' (he/she did) → Instead: 'उनमाथि आरोप छ कि'
- **Frame as:** "अनुसन्धानअनुसार" (According to investigation) or "आरोपअनुसार" (According to allegations)
- **Acknowledge:** "आरोपित व्यक्तिले सबै आरोप अस्वीकार गरेका छन्।" or similar
- **No inflammatory language** — No character judgments or assumptions

### Pause Mechanics
- **Standard pause:** [पज (२-३ सेकेन्ड)] marks natural breath points
- **Placement:** After major statements, before new ideas, between sections
- **Reading test:** Script should feel naturally paced, not rushed or dragging

### Video Production Cues (Reels Only)
- **[B-ROLL: description]** — What video should show during this audio
- **[GRAPHICS: description]** — Text overlays, animations, or visual emphasis
- **[पज]** — Pause length (let visuals land)
- **Timing:** Account for visual transitions; each section should have complementary b-roll ready

## Workflow

### Step 1: Ask Format & Length (PRIMARY)
**ALWAYS ask these first before anything else:**
- What is the script for? (Presentation or Reels/Voiceover)
- What length? (15 min, 20 min for Presentation OR 60-150 sec for Reels)

### Step 2: Gather Case Details (SECONDARY)
Only after format and length are confirmed:
- Case number, sector, amount, accused name(s), position(s)

### Step 3: Ask Format-Specific Details (TERTIARY)
**For Presentation:** Complexity level, number of presenters, additional context
**For Reels:** Main hook/angle, focus area for engagement

### Step 4: Assess Complexity
Determine if case is simple (1-2 accused, 2-3 allegations) or complex (3+, 4+).
(Only if not already determined in Step 3)

### Step 5: Generate Structure
For **Presentation**: Build flexible segment structure based on complexity and length.
For **Reels**: Create high-impact hook + evidence flow scaled to desired length.

### Step 6: Write Script in Nepali
Follow the appropriate structure from the use case sections above, using plain Nepali only.

### Step 7: Verify Constraints
Checklist:
- ✓ Nepali language only (no Hindi)
- ✓ Uses 'आरोप' consistently
- ✓ No assumed guilt language
- ✓ [पज] pauses marked (2-3 sec)
- ✓ For Reels: [B-ROLL] and [GRAPHICS] marked
- ✓ For Presentation: Presenter transitions clear + Q&A pauses marked
- ✓ Case number in title
- ✓ Reads naturally aloud
- ✓ Presumption of innocence maintained
- ✓ Correct length (within tolerance)

### Step 8: Delivery
Present the script in clear, readable format with section labels and timing notes.

## Notes for Claude

- **Two distinct workflows:** Ask user which format first
- **Elicitation is essential:** Don't skip the clarification questions
- **Presentation structure is flexible:** Adjust section count based on case complexity
- **Reels need visual planning:** [B-ROLL] and [GRAPHICS] must be specific enough for video team
- **Pauses are critical:** [पज] markers keep pacing natural; 2-3 seconds is standard
- **Nepali-only constraint:** No Hindi words; this is non-negotiable
- **Presumption of innocence:** Every script—no exceptions
- **Quality depends on:**
  - Clear, conversational Nepali
  - Consistent use of 'आरोप' (no assumed guilt)
  - Strategic pause placement
  - For Reels: Strong hook + compelling visuals
  - For Presentation: Clear presenter transitions + engagement points
- **Test by reading aloud:** Should sound natural, not rushed
- **If details sparse:** Ask for clarification rather than inferring
