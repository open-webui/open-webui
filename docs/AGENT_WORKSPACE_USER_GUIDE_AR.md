# دليل الاستخدام المختصر — Agent Workspace

## 1) التشغيل

- البيئة الأساسية: `docker-compose.yaml` (المنفذ `3001`).
- البيئة الداعمة (sidecar/search): `docker-compose.day1.yaml`.
- افتح الواجهة: `http://localhost:3001`.

## 2) اختيار النموذج

- المزود الافتراضي: OpenAI-compatible عبر sidecar.
- النموذج الافتراضي: `google/gemini-3-flash-preview`.
- يمكن تغيير النموذج من إعدادات النماذج داخل Open WebUI.

## 3) استخدام البحث الحي

- البحث الحي مفعل ومقيد بحوكمة النتائج.
- عند تعطل المزود أو انعدام النتائج: يتم إرجاع fallback مهذب (بدل خطأ قاسي) طالما:
  - `WEB_SEARCH_ENABLE_FALLBACK=true`

## 4) استخدام الملفات

- التخزين المعتمد: `local`.
- تدفق الاستخدام:
  1. رفع ملف من المحادثة.
  2. معالجة/تلخيص.
  3. إعادة استخدام نفس الملف في محادثات لاحقة.

## 5) Personas المقترحة

- **Researcher**: جمع مصادر حديثة + روابط.
- **Analyst**: تحليل النتائج واستخراج الاستنتاجات.
- **Executor**: تحويل التحليل إلى خطوات تنفيذ ومخرجات عملية.

> القوالب الجاهزة موجودة في `docs/AGENT_WORKSPACE_TEMPLATES_AR.md`.

## 6) المراقبة والحوكمة

- حوكمة التوكن/الميزانية: عبر متغيرات guardrail في `.env`.
- مراقبة الطلبات: اختبارات observability مفعّلة ومتحققة.
- تقرير التحقق النهائي: `VALIDATION_REPORT_AGENT_WORKSPACE_AR.md`.
