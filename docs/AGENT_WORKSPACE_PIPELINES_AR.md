# تعريف Pipelines التشغيلية — Agent Workspace

## Pipeline 1: ابحث → حلل → اكتب تقرير → احفظ

### مدخلات Pipeline 1

- سؤال/موضوع المستخدم.
- نطاق البحث (اختياري): عام/تقني/مالي.

### خطوات Pipeline 1

1. **Search**: تنفيذ بحث حي واسترجاع مصادر موثقة.
2. **Analyze**: تنقية النتائج + دمج + استنتاجات.
3. **Report**: كتابة تقرير منظم (ملخص، نقاط رئيسية، مخاطر، توصيات).
4. **Persist**: حفظ الناتج داخل المحادثة/ملاحظة قابلة للرجوع.

### مخرجات Pipeline 1

- تقرير مختصر قابل للتنفيذ.
- قائمة مصادر نهائية.

---

## Pipeline 2: اقرأ ملف → استخرج KPI → ملخص تنفيذي

### مدخلات Pipeline 2

- ملف (PDF/CSV/TXT).
- قائمة KPI مطلوبة (اختياري).

### خطوات Pipeline 2

1. **Ingest**: رفع الملف ومعالجته.
2. **Extract**: استخراج الأرقام/المؤشرات الأساسية.
3. **Summarize**: إنتاج Executive Summary موجه للقرار.
4. **Persist**: حفظ النسخة النهائية للاستخدام لاحقًا.

### مخرجات Pipeline 2

- جدول KPI واضح.
- ملخص تنفيذي منظم.

---

## ملاحظات تشغيلية

- في حال فشل البحث الحي، النظام يستخدم fallback مهذب إذا كان مفعّلًا.
- يمكن دمج pipeline مع Personas حسب السياق (Researcher ثم Analyst ثم Executor).
